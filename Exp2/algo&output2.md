Experiment – 02

**Algorithm**



STEP1: Start the program.



STEP2: Use a for loop to repeat the process for 3 players.



STEP3: Display a message asking to enter player details.



STEP4: Input the following data from the user:



* Player name



* Runs scored



* Number of balls faced



* Number of wickets taken



* Runs conceded
* Overs bowled
* Number of catches taken



STEP5: Calculate the strike rate using:

strike\_rate = (runs / balls) \* 100.



STEP6: Calculate the economy rate using:

economy = runs\_conceded / overs.



STEP7: Determine the batter performance using conditions:



* If runs ≥ 50 and strike rate ≥ 120 → Excellent batter
* If runs ≥ 30 and strike rate ≥ 100 → Good batter
* If runs ≥ 20 → Average batter
* Otherwise → Poor batter



STEP8: Determine the bowler performance:



* If wickets ≥ 3 and economy ≤ 6 → Excellent bowler
* If wickets ≥ 2 and economy ≤ 8 → Good bowler
* If wickets ≥ 1 → Average bowler
* Otherwise → Poor bowler



STEP9: Determine the fielder performance:



* If catches ≥ 2 → Outstanding fielder
* If catches = 1 → Active fielder
* Otherwise → Need improvement



STEP10: Determine the all-rounder performance:



* Excellent batter and Excellent bowler → Star all-rounder
* Good batter and Good bowler → Strong all-rounder
* Good batter or Good bowler → Supporting all-rounder
* Otherwise → Need improvement



STEP11: Display the player name, strike rate, economy, batter, bowler, fielder, and all-rounder decision.



STEP12: Repeat the process for all players.



STEP13: Stop the program.
